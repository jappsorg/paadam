# Firestore Database Schema for Interactive Worksheets

This document outlines the Firestore database structure designed to support Duolingo-like interactive worksheets, allowing users to fill out worksheets question by question and check answers as they go. This schema incorporates denormalization to optimize reads when a user resumes an incomplete worksheet.

## Collections

### 1. `users` Collection

- **Purpose:** Stores user profile information.
- **Document ID:** `user.uid` (string, from Firebase Authentication)
- **Fields:**
  - `email` (string): User's email address.
  - `displayName` (string): User's display name.
  - `photoURL` (string): URL to the user's profile picture.
  - `createdAt` (timestamp): Timestamp of when the user profile was created.

### 2. `worksheets` Collection

- **Purpose:** Stores the master templates or definitions of worksheets (the "blueprints"). These are the canonical versions of worksheets that users can attempt.
- **Document ID:** Auto-generated by Firestore (or a custom unique ID if preferred).
- **Fields:**
  - `title` (string): The official title of the worksheet.
  - `description` (string, optional): A brief description of the worksheet's content or purpose.
  - `config` (object): Parameters used to generate or define this worksheet (e.g., `type: "math"`, `difficulty: "easy"`, `subject: "addition"`).
  - `questions` (array of objects): An ordered list of question definitions for this worksheet template.
    - `questionId` (string): A unique identifier for the question _within this worksheet template_ (e.g., "q1", "q2_alpha").
    - `questionText` (string/object): The actual content/prompt of the question. Could be a simple string or a more complex object for rich formatting.
    - `questionType` (string): The type of question, influencing UI rendering and answer validation (e.g., "multiple-choice", "fill-in-the-blank", "true-false", "matching").
    - `options` (array of strings/objects, optional): For question types like "multiple-choice", this array holds the possible answer choices.
    - `correctAnswer` (string/object): The correct solution to the question. The structure should match the expected `userAnswer` format.
    - `points` (number, optional): The number of points awarded for correctly answering this question. Defaults to 1 if not specified.
  - `createdBy` (string, optional): The `userId` of the user who created this worksheet template, or a system identifier if system-generated.
  - `createdAt` (timestamp): Timestamp of when this worksheet template was created.
  - `version` (number, optional): A version number for the worksheet template, useful if templates can be updated.

### 3. `worksheetAttempts` Collection

- **Purpose:** Tracks each instance of a user attempting a specific worksheet. It includes the user's progress and answers (denormalized for read efficiency).
- **Document ID:** Auto-generated by Firestore.
- **Fields:**
  - `userId` (string): Foreign key linking to the `users.uid` of the user making the attempt.
  - `worksheetId` (string): Foreign key linking to the `worksheets` document ID (the template being attempted).
  - `worksheetTitle` (string, denormalized): The title of the worksheet, copied from `worksheets.title` for easy display in user history lists without an extra read.
  - `status` (string): The overall status of this attempt (e.g., "not-started", "in-progress", "paused", "completed").
  - `score` (number, optional): The user's overall score for this attempt, calculated upon completion or dynamically.
  - `currentQuestionIndex` (number, optional): The 0-based index indicating which question in the `questionsAttemptData` array the user is currently viewing or last left off on. Helps in resuming.
  - `questionsAttemptData` (array of objects): Stores the user's attempt state for each question. This array mirrors the order of questions in the `worksheets` template's `questions` array.
    - `questionId` (string): Matches the `questionId` from the corresponding question in the `worksheets` template. Used to link attempt data back to the master question definition.
    - `userAnswer` (string/object, optional): The answer submitted by the user for this question.
    - `isCorrect` (boolean, optional): Indicates if the `userAnswer` was correct. Set after the user checks their answer.
    - `status` (string): The status of the user's attempt for this specific question (e.g., "unseen", "seen", "answered", "correct", "incorrect-attempt", "skipped").
    - `attemptsCount` (number): The number of times the user has tried to answer this specific question during this worksheet attempt.
    - `lastAttemptTimestamp` (timestamp, optional): Timestamp of when this specific question was last interacted with or answered by the user.
  - `timeSpentSeconds` (number, optional): Total active time, in seconds, spent by the user on this worksheet attempt.
  - `startedAt` (timestamp): Timestamp of when this specific attempt was initiated by the user.
  - `lastActivityAt` (timestamp): Timestamp of the last interaction (e.g., answering a question, pausing) with any part of this attempt. Useful for sorting active attempts.
  - `completedAt` (timestamp, optional): Timestamp of when the user marked this attempt as completed.

## Mermaid Diagram of the Schema

```mermaid
graph TD
    subgraph User Data
        Users[("`users` Collection<br>docId: userId<br>email<br>displayName<br>photoURL<br>createdAt")]
    end

    subgraph Worksheet Templates
        Worksheets[("`worksheets` Collection<br>docId: worksheetId<br>title<br>description<br>config<br>questions [<i>array</i>]<br>  - questionId<br>  - questionText<br>  - questionType<br>  - options<br>  - correctAnswer<br>  - points<br>createdBy<br>createdAt<br>version")]
    end

    subgraph User Attempts
        WorksheetAttempts[("`worksheetAttempts` Collection<br>docId: attemptId<br>userId (FK)<br>worksheetId (FK)<br>worksheetTitle (denormalized)<br>status<br>score<br>currentQuestionIndex<br>questionsAttemptData [<i>array</i>]<br>  - questionId (links to Worksheets.questions.questionId)<br>  - userAnswer<br>  - isCorrect<br>  - status<br>  - attemptsCount<br>  - lastAttemptTimestamp<br>timeSpentSeconds<br>startedAt<br>lastActivityAt<br>completedAt")]
    end

    Users -- "One User has multiple" --> WorksheetAttempts;
    Worksheets -- "One Template has multiple" --> WorksheetAttempts;
```
